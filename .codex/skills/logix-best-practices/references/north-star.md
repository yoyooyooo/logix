# 统一北极星与裁决优先级

## 1) 北极星

- 单一事实源：`Static IR + Dynamic Trace`。
- 运行时优先：所有高层抽象最终都要回落到统一 runtime 语义。
- 确定性优先：能静态表达的协作关系不走黑盒。
- 证据先于结论：性能与诊断是交付物，不是附属品。

## 2) 裁决优先级（冲突时按此顺序）

1. 事务与语义安全（不破坏运行时不变量）。
2. 单一事实源（不引入并行真相源）。
3. 可解释性（诊断、追踪、回放可闭环）。
4. 工程可维护性（目录与职责边界清晰）。
5. 开发效率（模板/脚手架/局部捷径）。

## 3) 默认策略

- 跨模块协作：`linkDeclarative-first`，`link` 仅用于 async/external bridge。
- 状态更新：字段级写入优先（`mutate/immerReducers`）。
- 逻辑组织：显式 `setup/run`，并行 watcher 用 `Effect.all`。
- 依赖注入：Tag-only 契约 + 组合层 Layer 提供实现。

## 4) 违规信号（出现即回滚到北极星）

- 事务内出现 IO/嵌套 dispatch/`run*Task`。
- 同步联动使用 blackbox 协作且无法解释 tick 行为。
- 诊断事件不可序列化或体积失控。
- 同一能力在静态与动态层出现互相冲突的事实源。
