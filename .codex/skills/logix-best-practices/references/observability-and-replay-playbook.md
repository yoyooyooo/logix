# 可观测性与回放手册

## 1) 最小目标

- 事件可解释：能回答“为什么发生”和“由谁触发”。
- 事件可序列化：载荷 slim、结构稳定、可导出。
- 回放可验证：关键决策可以复现，不只是最终状态一致。

## 2) 事件分层

- 业务事件：action/state/process 级别的输入输出。
- 诊断事件：违规、降级、预算超限、guard 触发。
- 轨迹事件：运行时调度、事务、并发与协作边界。

## 3) 关键约束

- 诊断载荷使用可序列化值类型，避免把大对象直接塞进事件。
- 关键链路保留稳定标识（`instanceId/txnSeq/opSeq`）。
- 记录“边界事件”，尤其是事务窗口、best-effort 协作、降级路径。

## 4) 回放模型

1. 导出 evidence package（事件 + 元数据 + 版本信息）。
2. 按稳定标识重建执行顺序。
3. 对照静态控制面验证关键决策点。
4. 输出 replay 差异与不可回放原因。

## 5) 验收清单

- 关键诊断码有“触发/不触发”结论。
- 事件体积与频率满足预算，不因调试能力拖垮热路径。
- evidence package 可在独立环境读取并做最小回放。
- 回放结果可定位到失败根因，而不是停留在“表现异常”。
