# 测试策略手册

## 1) 分层矩阵

- L1 模块逻辑：state/actions/setup-run 合约。
- L2 并发与协作：watcher、task、process 语义。
- L3 事务与时间：事务窗口、异步边界、顺序保证。
- L4 宿主集成：React hooks / CLI / Worker 入口行为。
- L5 核心回归门：质量门 + 性能/诊断证据。

## 2) 每层最低断言

- L1：相位约束正确，纯更新路径不退化。
- L2：并发模式行为可预测（latest/exhaust/parallel）。
- L3：事务窗口内无 IO/异步逃逸，guard 行为可解释。
- L4：宿主 API 使用边界明确（provider/scope/lifecycle）。
- L5：核心路径改动有可复现 before/after 证据。

## 3) 测试风格

- 优先非 watch 一次性测试命令，避免阻塞会话。
- 对高风险语义用行为断言，不只看快照。
- 并发场景至少覆盖 happy path + 取消/中断 + 超时/失败。

## 4) 典型反模式

- 用“单个集成测试”替代分层验证。
- 在事务语义测试里引入不受控定时器。
- 只测成功路径，不测 guard 与降级路径。

## 5) 交付门槛

- 至少通过类型检查、lint、非 watch 测试。
- 核心路径改动必须附性能 diff 与诊断结论。
- 新增语义必须有可回归测试锚点（后续不靠人工记忆）。
