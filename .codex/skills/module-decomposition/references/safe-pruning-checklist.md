# 无损修剪检查清单

## 规划阶段（Plan）

- 是否明确拆分目的：行数治理 / 职责治理 / 子系统抽象（二选一：平铺 or 目录）？
- 是否把拆分写进某个“事实源载体”：包含候选文件清单（含 LOC）与互斥子模块边界？
- 是否明确每个子模块的单一职责（SRP）与依赖方向（单向拓扑）？
- 是否明确原文件保留内容（通常只保留顶层组装与对外导出）？

## 实施阶段（Implement）

- 是否保持对外导出与运行语义不变（无兼容层变更除外）？
- 是否避免在拆分过程中“顺手优化语义”（新缓存/新调度/新副作用）？
- 是否避免产生双真相源（同一协议/事件/状态定义复制到两处）？
- 是否避免循环依赖；若出现循环，是否回到边界设计而非硬 hack？

## 验证与交接（Verify / Handoff）

- 是否至少完成类型检查与相关测试（按项目约定）？
- 是否把“已拆/未拆/下一处拆分”回写到所选载体（文档/任务/PR/handoff 等）以便交棒？
