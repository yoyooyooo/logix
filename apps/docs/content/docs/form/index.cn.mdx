`@logixjs/form` 是 Logix 的输入状态领域包。

它定义这些领域语义：

- 可编辑的 values
- 校验与错误落点
- submit gating 与 decode
- 列表结构编辑与 row ownership

React 消费继续通过 `RuntimeProvider`、`useModule(...)` 和 `useSelector(...)` 回到 core host law。

## 范围

当前公开主线保持收口：

- 用 `Form.make(...)` 声明领域语义
- 通过 core React host 挂载
- 用 `useSelector(...)` 读取
- 通过 form handle 写入

selector support facts、field explanation 与 row-owned companion read 继续走同一条 `useSelector(...)` host route。
Form 不为这些读取新增 Form-owned React hooks、public path object、public row owner token 或 generic fact namespace。

当前默认 narrative 继续保持：

- quick-start
- instance lifecycle
- source / remote fact
- companion / local support fact
- field arrays / row identity

额外的 wrapper family、package-local React helper，以及另一套 form-specific hook family，都不在当前 canonical surface 内。
