`@logixjs/form` is the Logix domain package for editable input state.

It defines the semantics of:

- editable values
- validation and error placement
- submit gating and decode
- list structure editing and row ownership

React consumption continues to follow the core host law through `RuntimeProvider`, `useModule(...)`, and `useSelector(...)`.

## Scope

The current public route is intentionally small:

- author with `Form.make(...)`
- mount through the core React host
- read with `useSelector(...)`
- write through the form handle

Selector support facts, field explanations, and row-owned companion reads stay on the same `useSelector(...)` host route.
Form does not add Form-owned React hooks, public path objects, public row owner tokens, or generic fact namespaces for these reads.

The default narrative order remains:

- quick start
- instance lifecycle
- source / remote fact
- companion / local support fact
- field arrays / row identity

Additional wrapper families, package-local React helpers, and alternative form-specific hook families are outside the current canonical surface.
