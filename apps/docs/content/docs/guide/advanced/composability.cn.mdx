---
title: Composability
description: 通过 program imports、service layers 与普通 React 组合来组合 Logix。
---

组合是显式的。Logix program 可以 import 另一个 program，runtime 可以提供 service，React 可以围绕获取到的实例组合视图。

## Program imports

```ts
const ChildProgram = Logix.Program.make(Child, { initial: childInitial })

const ParentProgram = Logix.Program.make(Parent, {
  initial: parentInitial,
  capabilities: { imports: [ChildProgram] },
})
```

在 parent logic 中使用 child module tag。

```ts
const child = yield* $.use(Child.tag)
yield* child.actions.refresh()
```

## Service layers

基础设施用 layer。layer 不成为 module state。

```ts
const runtime = Logix.Runtime.make(AppProgram, {
  layer: Layer.mergeAll(ApiLive, LoggerLive),
})
```

## React composition

视图照常用 React 组合。每个组件只获取自己需要的实例。

```tsx
function ChildPanel() {
  const child = useModule(Child.tag)
  return <Panel value={useSelector(child, fieldValue("value"))} />
}
```

## 边界

不要写隐藏 tag、program 或 service 路线的 wrapper。helper 应该能机械还原到 program imports、layers 或 React components。
