---
title: Composability
description: Compose Logix by program imports, service layers, and ordinary React composition.
---

Composition is explicit. A Logix program can import another program, a runtime can provide services, and React can compose views around acquired instances.

## Program imports

```ts
const ChildProgram = Logix.Program.make(Child, { initial: childInitial })

const ParentProgram = Logix.Program.make(Parent, {
  initial: parentInitial,
  capabilities: { imports: [ChildProgram] },
})
```

Inside parent logic, use the child module tag.

```ts
const child = yield* $.use(Child.tag)
yield* child.actions.refresh()
```

## Service layers

Use layers for infrastructure. Layers do not become module state.

```ts
const runtime = Logix.Runtime.make(AppProgram, {
  layer: Layer.mergeAll(ApiLive, LoggerLive),
})
```

## React composition

Render components normally. Acquire only the instance each component needs.

```tsx
function ChildPanel() {
  const child = useModule(Child.tag)
  return <Panel value={useSelector(child, fieldValue("value"))} />
}
```

## Boundary

Do not create wrapper APIs that hide whether they use a tag, a program, or a service. Helpers should be mechanically reducible to program imports, layers, or React components.
