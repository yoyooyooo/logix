The guide follows the same shape as a Logix application: define a module, assemble a program, run it, project it into React, then add patterns and control planes only where the application needs them.

## Route

| Area | Pages |
| --- | --- |
| Get Started | [Introduction](/docs/guide/get-started/introduction), [Quick Start](/docs/guide/get-started/quick-start), search tutorials. |
| Essentials | [Canonical spine](/docs/guide/essentials/canonical-spine), state, effects, React, lifecycle, tick/flush. |
| Patterns | Search/detail, pagination, optimistic updates, i18n, form wizards. |
| Advanced | Concurrency, Suspense, testing, performance, scope, troubleshooting. |

Each section keeps the same object roles. `Module` defines, `Program` assembles, `Runtime` executes, React reads through `useModule` and `useSelector`.
