---
title: 国际化
description: 使用 service-first i18n 与 message token，不创建第二套 runtime 模型。
---

i18n 要么属于 runtime 环境，要么属于领域 message token。不要在 reducer 里硬编码翻译副作用。

## Service-first 路线

```ts
const logic = Module.logic("language", ($) =>
  Effect.gen(function* () {
    const i18n = yield* $.use(I18nService)
    yield* $.onAction("languageChanged").runLatest((action) => i18n.changeLanguage(action.payload))
  }),
)
```

通过 layer 提供 i18n service。组件仍然通过应用自己的 i18n adapter 渲染翻译结果。

## Message tokens

领域包可以存 message token，而不是存翻译后的字符串。例如 Form rules 可以返回稳定 error leaf，UI renderer 之后再翻译。

## 边界

runtime state 不应成为整份翻译 catalog 的副本。存 language、key 或 message token；渲染 adapter 留在 UI 边界。
