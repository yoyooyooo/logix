---
title: Internationalization
description: Use service-first i18n and message tokens without creating a second runtime model.
---

i18n belongs either to the runtime environment or to domain message tokens. Do not hard-code translation side effects inside reducers.

## Service-first route

```ts
const logic = Module.logic("language", ($) =>
  Effect.gen(function* () {
    const i18n = yield* $.use(I18nService)
    yield* $.onAction("languageChanged").runLatest((action) => i18n.changeLanguage(action.payload))
  }),
)
```

Provide the i18n service through a layer. Components still render the translated result through the app's normal i18n adapter.

## Message tokens

Domain packages may store message tokens instead of translated strings. Form rules, for example, can return stable error leaves that a UI renderer translates later.

## Boundary

Runtime state should not become a copy of the entire translation catalog. Store language, keys, or message tokens; keep rendering adapters at the UI edge.
