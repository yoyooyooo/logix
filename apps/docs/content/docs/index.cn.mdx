Logix 是 Effect-native 的应用逻辑运行时。React 继续负责渲染和组件组合；Logix 负责声明、装配、执行和运行时证据。

```text
Module.logic(...)        定义逻辑与局部声明
Program.make(...)        装配业务单元
Runtime.make(...)        创建执行容器
RuntimeProvider          把 runtime 投影到 React
useModule + useSelector  获取实例并精确读取
```

## 稳定路线

先读 [快速开始](/cn/docs/guide/get-started/quick-start)，再读 [Canonical spine](/cn/docs/guide/essentials/canonical-spine) 和 [React 集成](/cn/docs/guide/essentials/react-integration)。后续页面都沿着同一组对象展开，不引入第二套模型。

用户文档不把内部编译器名词放到主路线。字段声明通过 logic builder 或 Form 这类领域 DSL 编写；运行时控制面统一停在 `Runtime.check` 与 `Runtime.trial`。
