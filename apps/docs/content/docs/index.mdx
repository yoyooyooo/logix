Logix is an Effect-native runtime for application logic. React keeps rendering and component composition; Logix owns declaration, assembly, execution, and runtime evidence.

```text
Module.logic(...)        define logic and local declarations
Program.make(...)        assemble the business unit
Runtime.make(...)        create the execution container
RuntimeProvider          project the runtime into React
useModule + useSelector  acquire an instance and read it precisely
```

## Stable route

Start with [Quick Start](/docs/guide/get-started/quick-start), then read [Canonical spine](/docs/guide/essentials/canonical-spine) and [React integration](/docs/guide/essentials/react-integration). The rest of the guide expands those same objects instead of introducing a second model.

Logix deliberately keeps internal compiler names out of the user route. Field declarations are authored through the logic builder or through domain DSLs such as Form; runtime control-plane reports stay under `Runtime.check` and `Runtime.trial`.
