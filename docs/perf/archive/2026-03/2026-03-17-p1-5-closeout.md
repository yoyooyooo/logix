# 2026-03-17 · P1-5 closeout：selector activation retention 线默认关闭

## 结果分类

- `accepted_with_evidence`

## 结论

`P1-5` 这条线到当前为止已经收口，默认不再继续开实现刀。

当前已接受的切口：

- React selector activation grace hold
- core selector retain/release 下沉
- retain 冷路径 scope 税清理
- selector entry / hub long-gap 复用
- readQuery store idle GC
- cached entry cap + value reset

当前仅保留 evidence-only 的候选：

- `explicit cached entry eviction`

### 1. activation restart 已经归零

- `useSelector.sharedActivation.microbench.ts`

## 为什么默认关闭

- corrected retained heap 已接近平台期
- `explicit cached entry eviction` 没有拿到比第六刀更硬的收益
- 继续压榨的边际收益已经明显小于新增复杂度

## 后续开启条件

只有在以下情况之一出现时，才应该重新打开 `P1-5`：

- browser / node retained heap gate 再次出现稳定 blocker
- selector activation 相关路径出现新的回归证据
- 有新的产品级内存预算要求，且必须把 `~1MB` 级常驻继续向下压

否则默认动作是：

- 不再继续改 `P1-5`
- 把精力切回新的 current-head blocker 或新候选线
