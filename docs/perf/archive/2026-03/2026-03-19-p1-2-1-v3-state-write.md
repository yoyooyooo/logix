## 目标与边界

## 代码改动

- `packages/logix-core/src/internal/runtime/core/StateTransaction.ts`
  - `recordKnownTopLevelDirtyEvidence` 放开 top-level list root。
  - 命中 list root 时记录 `${key}[]`，不再直接放弃并回退 whole-state 路径。
- `packages/logix-core/src/internal/runtime/core/BoundApiRuntime.ts`
  - `state.update` 去掉 list root 直接回退 `runtime.setState` 的守门。
  - 对 top-level change 统一记录可追踪 patch，list root 走 `${key}[]`。
- `packages/logix-core/src/internal/runtime/core/ModuleRuntime.dispatch.ts`
  - 保持逻辑不变，补注释说明 reducer/action-writeback 已可覆盖 top-level list root evidence。
- `packages/logix-core/src/internal/state-trait/external-store.ts`
  - 新增 `resolveExternalStorePatchPath`，优先规范化路径，规范化失败时保留 raw string 给 `StateTransaction` 解析，减少非必要 fallback。
- `packages/logix-core/test/internal/Runtime/ModuleRuntime/ModuleRuntime.reducerWholeStateFallback.Perf.off.test.ts`
  - 新增 `listRoot` case，验证 list root 场景也能走 top-level evidence。
