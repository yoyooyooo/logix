# 2026-03-19 · P2-4（runtime devtools / observability plane）下一刀

## 现状复核（以已吸收的 P2-4A/B 为前提）

P2-4A 与 P2-4B 已把 core 侧的两条证据链补齐到可导出形态：

## 还剩的未完成切口

Devtools React 侧仍缺一个“可见性闭环”切口：

- live snapshot 的 `exportBudget.byEvent` 目前没有稳定展示入口，导致预算热点仍只能在离线 JSON 里看。
- evidence import 流程此前会丢失 `summary.projectionBudget` 的可读入口，导入后也很难在 UI 内快速定位热点。

这条线的 ROI 来自“把已存在的证据链变成可读诊断”。

## 本 worktree 的最小 slice（已实现）

只在 evidence import 路径做最小可见性增强，不碰 dispatch/state write 主路径，不增加重 payload：

- 解析 `evidence.summary.projectionBudget`。
- 在导入后的 `snapshot.events` 头部插入 1 条合成事件 `devtools:projectionBudget`，把 `totals + Top10 byEvent` 作为 `meta` 暴露到既有 Timeline/Inspector 链路。
- 同步填充导入态 snapshot 的 `exportBudget`（包含 `byEvent` Map），保证离线视角与 live 结构一致。

## 下一步建议
