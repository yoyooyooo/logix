# Research

本目录用于收集“可复现的调研结论/对比/选型依据”，不要求与代码实现严格同步，但要求：

- 结论可追溯到具体仓库/文件/commit（至少到仓库 + 关键文件路径）
- 明确适用范围与已验证边界（例如：仅示例工程 / 可直接运行 / 仅库不含服务端入口）

## Topics

- `effect-ts-backend-open-source`：开源 Effect（effect-ts）后端项目写法调研
- `effect-ts-ai-agent-workflow-landscape`：基于 Effect（effect-ts）的 AI agent / workflow 开源生态调研
- `effect-ts-state-management-landscape`：基于 Effect（effect-ts）的状态管理 / 副作用管理上层封装调研
- `spec-driven-tools-landscape`：Spec 驱动上层工具/平台版图与机会窗口调研
