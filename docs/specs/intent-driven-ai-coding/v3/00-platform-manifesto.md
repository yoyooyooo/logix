> **核心定位**：Intent-Driven AI Coding 平台是前端开发者的**意图显影引擎 (Intent Development Engine)**。它帮助开发者将脑海中模糊的业务意图，通过结构化的显影过程，转化为精确的高质量代码。

## 1. 核心哲学：思维的外骨架

在编码之前，开发者实际上已经在脑海中完成了“需求分析”和“架构设计”。本平台旨在将这一隐性过程**显性化**：

*   **显影 (Development)**：不要直接跳进代码细节。先在 Spec 层定义“要做什么 (What)”，再让 AI 帮你生成“怎么做 (How)”。
*   **连续体 (Continuum)**：Spec（需求/设计）和 Impl（代码）是同一工作的两个阶段。在平台上，你可以无缝切换“设计师”和“工程师”的双重身份。
*   **降熵 (Entropy Reduction)**：利用 LLM 处理从 Spec 到 Impl 的繁琐转化，让你专注于核心逻辑的设计。

## 2. 三位一体模型 (The Trinity)

我们信仰**极简主义**。无论你是画原型还是写代码，世界都由三个维度构成：

1.  **UI (Presentation)**：**躯壳**。界面结构与视觉表现。
2.  **Logic (Behavior)**：**灵魂**。业务规则与流程编排。
3.  **Domain (Data)**：**记忆**。业务概念与数据模型。

## 3. 工作流：一人分饰两角 (Solo Workflow)

在前期，前端开发者是平台的**超级用户**，同时掌控 Spec 和 Impl：

*   **设计态 (Spec Mode)**：
    *   *动作*：贴入 Figma 截图，写下逻辑大纲 (User Story)，定义业务名词。
    *   *价值*：**理清思路**。在写代码前，先确保自己想清楚了。
*   **实现态 (Impl Mode)**：
    *   *动作*：一键生成组件树、Flow DSL 和 Schema，然后在 Studio 中精修。
    *   *价值*：**提效**。让 AI 完成 80% 的样板工作，你只负责 20% 的核心逻辑。
*   **LLM 的角色 (Co-Pilot)**：
    *   它是你的**结对编程伙伴**。你告诉它 Spec，它给你 Impl。你修改 Spec，它帮你重构 Impl。

## 4. 视图体系

*   **Doc View**：快速录入你的想法和逻辑大纲。
*   **Canvas View**：可视化梳理组件结构和流程拓扑。
*   **Studio View**：深入代码细节，进行精确控制。

> **未来演进**：当团队成熟后，可以将 Spec Mode 的权限开放给 PM 和设计师，实现真正的全链路协作。但在那之前，这是你个人的**超级 IDE**。
