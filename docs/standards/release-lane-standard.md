# Release Lane Standard

## Current Lane

`main` is the active future line for Logix development and release preparation.

The current public packages still depend on `effect@4.0.0-beta.28`. Because the runtime API and dependency baseline are still moving, npm publishes should remain prerelease packages and use the `beta` dist-tag until the stable cutover gate is explicitly met.

## Release Scripts

- `pnpm ci:release:beta` is the current release verification gate.
- `pnpm release:beta` publishes through Changesets. In prerelease mode it publishes the prerelease versions already generated by Changesets. Outside prerelease mode it forces the npm `beta` dist-tag.
- `.github/workflows/release.yml` is the Trusted Publishing workflow and should run from `main`.

The script names still say `beta` because the current npm lane is beta. Do not rename them to `release` or `latest` until the stable cutover is actually performed.

## Stable Cutover Gate

Stable npm `latest` release requires all of the following:

```text
Effect v4 dependency decision accepted
public package versions exit prerelease semver
.changeset/pre.json removed by changeset pre exit
.changeset/config.json baseBranch is main
release workflow publishes without --tag beta
release CI passes build, typecheck, test, browser smoke, and required perf gates
README and package descriptions no longer imply beta-only usage
```

Stable cutover is a release event, not a branch rename. Merging `next-api` into `main` accelerates development and CI artifact collection, but it does not by itself make the npm packages release-safe or stable.

## Forbidden Shortcuts

- Do not publish beta dependency builds under `latest`.
- Do not treat a clean PR CI run as a release gate.
- Do not remove prerelease semver while `.changeset/pre.json` is still active.
- Do not let package descriptions claim stability that the runtime/dependency gate has not earned.

## Current Conclusion

Use `main` as the active development and release-preparation line. Keep npm prerelease publishing until the stable cutover gate is intentionally completed.
