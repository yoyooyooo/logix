# apps/docs 文档翻译过程中的“待升级点”备忘

> 目标：在翻译英文文档（以中文为基准）过程中，顺带做一次文档质量巡检：
>
> - 小问题：就地修（会同步到中英文或至少不破坏一致性）。
> - 大问题：先记录在此文件，等翻译阶段性完成后统一给出建议/改造方案再动手。

## 大问题（先记录，后统一提案）

1. **用户文档出现内部版本/里程碑代号**  
   例：`guide/advanced/txn-lanes.md` 中的“`062` 起默认开启”。需要确定用户文档的版本叙事策略（是否用“自某版本起/默认开启”而不暴露内部编号）。

2. **部分页面代码块（Tab/多段 code fence）可读性与结构偏脆**  
   例：`guide/get-started/tutorial-complex-list.md` 的 `tab="Logic DSL"/"Flow API"` 片段混合缩进与 fence，虽可编译但后续维护风险高。建议后续统一整理为“Progressive Examples”规范写法。

3. **术语与语言一致性在中文原文里有漂移**  
   例：部分中文页面标题/描述已使用英文（如 `Thinking in Logix`、`Modules & State`），但正文仍是中文且术语中英混排不统一。需要确定“中文用户文档”的术语策略：保留英文术语（并统一大小写）还是翻译为中文并在首次出现处标注英文。

## 小问题（随翻译过程直接修复）

- 先占位：逐页翻译时把明显的英文侧导航标题、链接文案、术语一致性等小问题就地修掉。
  - 已处理：`guide/learn/managing-state.md` 与 `guide/learn/managing-state.en.md` 中示例的 `yield *` → `yield*`（避免误导与语法噪音）。
