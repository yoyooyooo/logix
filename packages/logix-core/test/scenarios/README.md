# Core Scenarios

These scenarios represent the core boundary cases for the Logix runtime.

- **Optimistic.ts**: Optimistic UI updates with rollback on failure.
- **Cascading.ts**: Dependent state updates (e.g., cascading selects).
- **AsyncFlow.ts**: Long-running tasks (polling), cancellation, and debouncing.
- **Batch.ts**: Batch operations and transactional flows.
