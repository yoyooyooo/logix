# Quickstart: 015 Devtools Converge Performance Pane

**Feature**: `015-devtools-converge-performance`  
**Created**: 2025-12-18  

本 Quickstart 的目标是用最少步骤验证 015 的三个核心承诺：

1) converge 时间轴可用（多行 + txn 分段 + 关键标签）；  
2) Audits 可操作（有稳定 ID + 建议 + 可复制代码片段）；  
3) 证据不足不误导（显式降级），且内嵌面板与离线导入结论一致。

## 1) 准备一份包含 converge 证据的证据包

- 证据包必须包含 converge 相关的证据事件（至少能定位到事务窗口，并能识别 converge 相关事务）。
- 若证据包不包含 converge 证据：本面板应显示空态并提示如何获取证据包。

## 2) 离线导入验证（MVP 必做）

1. 导入证据包。
2. 打开 converge performance pane。
3. 选择一个模块实例，并框选一个时间窗口。
4. 验证：
   - 能看到 converge 相关事务的多行时间轴；
   - 每笔事务能看到关键标签：执行路径/原因/配置来源；
   - 至少产出一条审计条目（稳定 ID）与“建议 + 可复制代码片段”（Provider 覆盖优先）。

## 3) 内嵌面板（live）一致性验证（MVP 必做）

1. 在 live 模式打开同一运行的面板。
2. 对比离线导入的同一时间窗口：
   - converge 事务数量一致；
   - 排序一致（若缺失全局排序键，退回实例内单调序号）；
   - 审计命中一致（允许 UI 展示差异，但结论一致）。

## 4) 证据不足降级验证（MVP 必做）

1. 使用一份“缺失部分字段”的证据包（例如缺少更细粒度的耗时分段或热点细节）。
2. 验证：
   - 面板不崩溃、不白屏；
   - 审计条目会降级为 “证据不足”，并明确缺失字段；
   - 不会输出误导性的结论与代码片段。
