## 当前结论

- Gate-A / Gate-B：`PASS`（strict 证据链已齐备）。
- G1：`NOT_PASS`（strict perf/diagnostics 预算仍未通过）。
- 发布级最终裁决：继续以 GitHub `logix-perf-sweep.yml` 的 `perf_profile=soak + diff_mode=strict` artifact 为准。
- 新复测显示：`runtimeStore/externalStore` 在多轮中总体维持回收，但 `form.listScopeCheck@light` 仍跨轮失稳（含 `after=null`）。当前应继续聚焦 form 链路（auto/full 比值与 light 诊断开销），并以 repeated quick 的中位结论 + broad `soak+strict` 作为最终 gate 判定口径。
