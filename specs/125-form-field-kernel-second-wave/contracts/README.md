# Contracts: Form Field-Kernel Second Wave

## 1. Boundary Contract

- Form DSL 承接领域表达
- field-kernel 承接底层能力

## 2. Authoring Contract

- `logic` 家族是主方向
- `commands` 是明确辅入口
- direct API 不回顶层主线
- `Form.make / Form.from` 是顶层主入口
- `Form.derived` 不再作为顶层 barrel 入口

## 3. Naming Contract

- naming 未定不影响结构边界
- `FieldKernel` 继续作为内部符号名 / direct API 口径
