# 172 Conceptual DevTools Parity Matrix

本表是 172 的 route SSoT。它不追当前 DevTools UI，而是追“概念 DevTools 会问什么 Runtime 问题，CLI 是否有等价机器问法”。

Status 规则：

- `Must implement`: 172 必须实现 owner-backed payload。
- `Owner-backed or gap`: 172 必须提供 owner-backed payload 或 structured gap。
- `Deferred`: 172 只冻结 future owner、gap shape 和 reopen bar。
- `Rejected`: 172 不提供 route。

Artifact family 规则：

- inspect 类输出统一使用 `LiveInspectArtifact(section=...)`。
- mutation 输出仍使用 `LiveOperationFacet`。
- evidence handoff 输出仍使用 `CanonicalEvidencePackageRef`。
- target discovery 输出沿用 `LiveTargetIndex` / `LiveTargetDetail`。
