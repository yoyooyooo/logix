# Research: React Host Adjunct Evidence Closure

## Decision: React Host Evidence Is Adjunct And Subordinate

**Rationale**: Runtime owner facts explain declaration, runtime state, operations, fields and live evidence. React host evidence can explain selector/render symptoms and interaction linkage, but it must not override runtime truth or become verification authority.

**Alternatives considered**:

- Make host evidence a Runtime fact owner. Rejected because it would own timeline/order/state semantics outside Runtime owners.
- Keep host evidence out entirely. Rejected because real Agent UI diagnosis blind spots remain.

## Decision: Linkage Uses Runtime Coordinates And Approved Markers

**Rationale**: Target coordinate, attachment id, `txnSeq`, `opSeq`, `linkId`, artifact refs, gaps, redaction and degraded markers are already part of Runtime inspect coordination. Reusing them keeps host evidence traceable without creating new truth.

**Alternatives considered**:

- Use component names or DOM paths as primary identity. Rejected because they are host-local and unstable.
- Add a second selector identity. Rejected because `10-react-host-projection-boundary.md` already fixes selector law.

## Decision: Conflicts Produce Disagreement Markers, Runtime Truth Wins

**Rationale**: Host data can be delayed, missing, degraded or conflicting with runtime facts. A structural disagreement marker preserves diagnostic value while keeping runtime owner facts authoritative.

**Alternatives considered**:

- Let host evidence override runtime facts when fresher. Rejected because freshness does not equal owner authority.
- Drop conflicting host evidence silently. Rejected because Agents lose diagnostic context.

## Decision: Local Profile Summary Is Bounded Local-Only Evidence

**Rationale**: Local profile context can help diagnose cost symptoms, but raw profiles or unbounded samples would create profiler truth and memory risk. A summary with budget, redaction and runtime refs is sufficient for Agent diagnosis.

**Alternatives considered**:

- Export raw CPU/heap profiles. Rejected as out of scope and too heavy.
- Treat profile data as timeline facts. Rejected because profile summaries do not own Runtime ordering or state.

## Decision: Disabled Safety Is A Hard Gate

**Rationale**: Host instrumentation can easily tax render and subscription paths. Disabled mode must have no host capture buffer, no extra render subscription fanout and no transaction-window IO.

**Alternatives considered**:

- Allow always-on low-cost capture. Rejected unless owner law is reopened with measurable proof.
- Defer overhead validation to implementation polish. Rejected because it can invalidate React host performance boundaries.

## Decision: 182 Remains Stopped

**Rationale**: 188 is the adopted terminal path for host adjunct evidence. Reviving 182 as standalone owner would split authority and confuse active docs.

**Alternatives considered**:

- Reopen 182 and implement there. Rejected because current SSoT routes terminal diagnosis through 183/188.
- Keep both 182 and 188 active. Rejected because it creates parallel planning truth.
