# Specification Quality Checklist: Final Form/Kernel Single-Track Cutover

## Requirement Completeness

- [ ] Are all final owner lanes covered by explicit requirements?
- [ ] Are stale docs and compatibility code both covered, not only docs?
- [ ] Are source, companion, final truth, row/list, host read, verification, and performance gates all represented?
- [ ] Are required witnesses named with pass/fail evidence expectations?

## Clarity

- [ ] Is every forbidden route named explicitly?
- [ ] Is `Runtime.compare` wording distinguished from productized Form root compare?
- [ ] Is toolkit allowed only under mechanical-reduction proof?

## Measurability

- [ ] Does every requirement have a scanner, test, witness, or evidence artifact?
- [ ] Are quick perf artifacts excluded from hard success claims?

## Consistency

- [ ] Does this spec avoid compatibility/deprecation language?
- [ ] Do docs, source, tests, examples, and specs route to the same authority map?
